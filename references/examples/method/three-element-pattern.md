# Three-Element Pattern (Rationale / Procedure / Safeguard) — Worked Overview

`Use the three-element pattern to write any Methods subsection: state the rationale, the procedure, and the safeguard. The companion file walks through it block-by-block on a worked clinical example.`

## Mapping

1. **Rationale:** what the unresolved question is, why this design / measurement / analysis was chosen.
2. **Procedure:** what was done, in execution order.
3. **Safeguard:** how bias is controlled — blinding, prespecification, validation, sensitivity analyses.

## Local source references

1. Annotated walk-through of a Methods section: `references/examples/method/methods-section-walkthrough.md`
2. Worked subsection example: `references/examples/method/example-of-the-three-elements.md`

## Direct usage

1. Read `methods-section-walkthrough.md` to map each paragraph to one of the three roles.
2. Rewrite your own Methods subsection in the same role order.
3. Cross-check that each role is present and that the safeguards are tied to a measurable property where possible.
