# Novel-Question Decomposition (Worked Pattern)

`For a novel medical research question, decompose the difficulty into clear points so the rationale for the present design becomes self-evident.`

## Skeleton

```
% State the aim
Our aim is to [AIM] in [POPULATION].

% Point 1: define the substantive challenge
First, [observable difficulty] because [methodological / clinical reason].

% Point 2
Second, [observable difficulty] because [methodological / clinical reason].

% Point 3
Finally, [observable difficulty] because [methodological / clinical reason].
```

## Worked example (illustrative; fictional)

```
Our aim is to develop and externally validate a clinical prediction model for
30-day mortality after emergency laparotomy in patients aged 75 years or older.
This task is challenging for three reasons.

First, available risk scores (P-POSSUM, NELA, ACS-NSQIP) were derived in mixed-age
populations and underestimate frailty effects in older adults, leading to
miscalibration in the highest-risk band.

Second, frailty itself is heterogeneously measured across centres (Clinical Frailty
Scale, Edmonton Frail Scale, modified Frailty Index), and harmonization across
data sources is non-trivial.

Finally, external validation is rarely performed for prediction models in this
population, and where attempted, it is often constrained to a single national
audit dataset, which limits geographical generalizability.
```

## Notes

1. Use this pattern when introducing a prognostic model, a new diagnostic, or a new clinical entity.
2. Each point should be reflectively addressable in the Methods (e.g., we adopted the Clinical Frailty Scale and validated externally in two registries).
3. Keep the section to 4 to 6 sentences; reviewers do not reward length here.
